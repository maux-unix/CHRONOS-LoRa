62/* correct_convolutional_encode uses the given conv instance to

64 * encoded must be long enough to hold the resulting encoded length,

66 * However, this length should first be converted to bytes, as that
67 * function returns the length in bits.

70 * this is not an exact multiple of 8, then it occupies an additional

76/* correct_convolutional_decode uses the given conv instance to
77 * decode a block encoded by correct_convolutional_encode. This
78 * call can cope with some bits being corrupted. This function
79 * cannot detect if there are too many bits corrupted, however,
80 * and will still write a message even if it is not recovered
81 * correctly. It is up to the user to perform checksums or CRC
82 * in order to guarantee that the decoded message is intact.
83 *
84 * num_encoded_bits should contain the length of encoded in *bits*.
85 * This value need not be an exact multiple of 8. However,
86 * it must be a multiple of the inv_rate used to create
87 * the conv instance.
88 *
89 * This function writes the result to msg, which must be large
90 * enough to hold the decoded message. A good conservative size
91 * for this buffer is the number of encoded bits multiplied by the
92 * rate of the code, e.g. for a rate 1/2 code, divide by 2. This
93 * value should then be converted to bytes to find the correct
94 * length for msg.
95 *
96 * This function returns the number of bytes written to msg. If
97 * it fails, it returns -1.
98 */

183/* correct_reed_solomon_create allocates and initializes an
184 * encoder/decoder for a given reed solomon error correction
185 * code. The block size must be 255 bytes with 8-bit symbols.
186 *
187 * This block can repair corrupted bytes. It can handle as
188 * many as num_roots/2 bytes having corruption and still recover
189 * the encoded payload. However, using more num_roots
190 * adds more parity overhead and substantially increases
191 * the computational time for decoding.
192 *
193 * primitive_polynomial should be one of the given values in this
194 * file. Sane values for first_consecutive_root and
195 * generator_root_gap are 1 and 1. Not all combinations of
196 * values produce valid codes.
197 */

203/* correct_reed_solomon_encode uses the rs instance to encode
204 * parity information onto a block of data. msg_length should be
205 * no more than the payload size for one block e.g. no more
206 * than 223 for a (255, 223) code. Shorter blocks will be encoded
207 * with virtual padding where the padding is not emitted.
208 *
209 * encoded should be at least msg_length + parity length bytes long
210 *
211 * It is allowable for msg and encoded to be the same pointer. In
212 * that case, the parity bytes will be written after the msg bytes
213 * end.
214 *
215 * This function returns the number of bytes written to encoded.
216 */
